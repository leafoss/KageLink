Use as APIs apropriadas do Windows para obter o HWND e o retângulo da área cliente.

Se um método de captura não funcionar em determinada forma de exibição, implemente fallbacks seguros para janela normal, maximizada e tela cheia.

Não declare suporte a uma condição que não tenha sido realmente testada.

Se a janela estiver minimizada e não puder ser capturada corretamente, apresente no aplicativo uma mensagem clara, sem afetar o chat.

==================================================
6. TRANSMISSÃO DA IMAGEM
==================================================

Esta é uma V1 leve e estável. Não implemente WebRTC nesta etapa.

Configuração-base:

- resolução transmitida: 960 × 540;
- proporção: 16:9;
- FPS pretendido: entre 8 e 12;
- sem áudio;
- baixa latência;
- foco em estabilidade e baixo consumo.

Preferencialmente, utilize:

- WebSocket separado para a transmissão;
- quadros JPEG ou WebP em formato binário;
- compressão adequada para manter 8 a 12 FPS;
- descarte de quadros antigos;
- exibição sempre do quadro mais recente.

Não crie uma fila grande de imagens, porque isso aumentaria o atraso.

A transmissão da aba GAME deve utilizar o mesmo:

- endereço do servidor;
- domínio;
- túnel seguro;
- token de autenticação;
- processo de conexão já utilizado pelo KageLink.

O módulo GAME pode e deve possuir endpoints e WebSockets próprios, separados dos canais atuais de chat.

Exemplo de separação arquitetural:

- sistema atual de chat: permanecer intocado;
- canal novo para stream do jogo;
- canal novo para comandos do jogo;
- endpoint novo para status da janela e do stream.

Os nomes exatos das rotas podem seguir o padrão existente do projeto.

==================================================
8. JOYSTICK
==================================================

Adicionar um joystick virtual transparente no lado inferior esquerdo.

O joystick deve controlar as teclas direcionais do teclado:

- cima = seta para cima;
- baixo = seta para baixo;
- esquerda = seta para esquerda;
- direita = seta para direita.

Permitir diagonais utilizando duas teclas simultaneamente:

- cima + direita;
- cima + esquerda;
- baixo + direita;
- baixo + esquerda.

O joystick poderá ser visualmente analógico, mas sua saída deve ser digital em oito direções.

Funcionamento:

- ao mover o joystick para uma direção, enviar key-down da tecla correspondente;
- ao mudar de direção, soltar as teclas que deixaram de ser necessárias;
- ao voltar o joystick ao centro, enviar key-up de todas as setas;
- não enviar centenas de comandos repetidos sem necessidade;
- enviar novos eventos apenas quando o estado das direções mudar.

Adicionar uma zona morta adequada para evitar movimentos involuntários.

O joystick deve aceitar pressionamento contínuo.

==================================================
9. BOTÕES A, B, C E D
==================================================

Adicionar quatro botões transparentes no lado inferior direito, conforme a disposição visual da imagem Idea.png.

Mapeamento fixo:

- A = tecla E;
- B = tecla Espaço;
- C = tecla G;
- D = tecla V.

O usuário não poderá alterar esse mapeamento nesta versão.

Todos os quatro botões devem aceitar:

- toque rápido;
- pressionamento contínuo.

Comportamento:

- pointer-down ou touch-down = key-down;
- pointer-up = key-up;
- pointer-cancel = key-up;
- perda de foco = key-up;
- saída da tela = key-up.

Não implementar toggle.

Um toque rápido deve resultar naturalmente em pressionar e soltar a tecla.

Um pressionamento contínuo deve manter a tecla pressionada enquanto o dedo permanecer sobre o botão.

O aplicativo deve possuir suporte real a multitouch, permitindo, por exemplo:

- manter o joystick para cima;
- manter a tecla E pressionada ao mesmo tempo;
- tocar em outro botão sem interromper o movimento.

Caso mensagens simples do Windows não sejam reconhecidas pelo jogo, implemente um fallback confiável utilizando as APIs apropriadas de entrada do Windows, mas sempre validando que a janela-alvo é “Shinobi Story Online”.

Não envie comandos para outra janela por engano.

Teste especificamente:

- pressionar diagonais;

- usar Espaço;

Implemente o gerenciamento de foco de forma segura e limitada ao jogo.
